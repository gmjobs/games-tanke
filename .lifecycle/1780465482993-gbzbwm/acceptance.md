# 坦克大战豪华版 —— 验收测试（Acceptance Tests / Definition of Done）

> 适用版本：豪华版 V1（含评审修订 2026/6/3）
> 视角：黑盒、用户视角，不涉及内部实现
> 说明：本验收基线 = 规格说明 AC + 已评审交互原型（主菜单 / 模式选择 / 设置 / 最高分 / 帮助 / HUD / 暂停 / 过关结算 / 游戏结束 / 触屏控件 / 大字号结算分数）。
> 评审修订落点：**游戏结束结算分数字体相对原型再放大 2 个字号**（详见 AT-G2 / 场景 G2）。

## 二、关键端到端场景（Gherkin）

```gherkin
Feature: 单人通关核心流程
功能：玩家从打开网页到通过一关的完整闭环（US-1, US-2）

  Scenario: 打开即玩并通过第一关
    Given 玩家首次打开「坦克大战·豪华版」网页
    And 页面已在 3 秒内加载完成
    When 页面加载完成
    Then 显示主菜单，含「开始游戏/模式选择/设置/最高分/操作说明」
    When 玩家点击「开始游戏」（单人模式）
    Then 进入第 1 关，战场显示玩家坦克、≥2 个敌人出生点和受保护基地
    And HUD 显示 生命数、当前分数、关卡=1、剩余敌人数=20、最高分
    When 玩家用方向键移动并用空格射击，击毁本关全部敌人
    And 基地始终未被命中
    Then 显示「第 1 关 通过！」结算，含按类型击毁统计与本关得分
    When 玩家点击「进入下一关」
    Then 进入第 2 关，且第 2 关敌人数量/强度或地形复杂度不低于第 1 关
```

```gherkin
Feature: 游戏失败结算与最高分（评审修订：结算分数大字号）
功能：基地被毁触发失败结算并刷新最高分（US-2, US-6, FR-8）

  Scenario: 基地被摧毁导致游戏结束并刷新纪录
    Given 玩家正在第 3 关作战
    And 当前累计得分高于历史最高分
    When 任意一发子弹命中基地
    Then 立即判定游戏结束（失败），显示「游戏结束」结算
    And 失败原因显示「基地被摧毁」
    And 结算总分以大字号显示，且该分数字体相对已评审原型再放大 2 个字号
    And 显示「★ 新纪录！★」提示并将本局总分写入 localStorage 作为新最高分
    When 玩家点击「返回主菜单」
    Then 回到主菜单，最高分入口显示刷新后的纪录
```

```gherkin
Feature: 双人本地合作
功能：两名玩家同键盘合作与各自计分（US-4, FR-10）

  Scenario: P1 阵亡后 P2 继续直至基地被毁
    Given 玩家在主菜单将模式切换为「双人本地合作」并开始游戏
```

## 三、覆盖路径说明（正常 / 边界 / 异常）

### 正常路径（Happy Path）
- AT-A1~A4、AT-C1~C5、AT-E1、AT-G1/G4、AT-H1~H3、AT-I1/I4、AT-J1~J2、AT-K1~K2、Gherkin「单人通关核心流程」「道具拾取」「双人合作」。

### 边界路径（Boundary）
| 场景 | 关注点 | 关联用例 |
|------|--------|----------|
| 同屏敌人达到上限 4 | 第 5 个敌人不刷新，待场上有空位才补 | AT-D1 |
| 子弹上限边界 | 默认上限 1 时第 2 发被抑制；拾火力后边界提升 | AT-C5 / AT-C6 |
| 最后一关通关 | 第 10 关清场显示「通关」而非普通过关 | AT-G3 |
| 限时道具到期瞬间 | 护盾/定身/基地加固到期临界，效果准确失效 | AT-F5~F7 |
| 当前分 == 历史最高 | 平分时是否刷新（按规格「刷新记录时提示」，相等不应误报新纪录） | AT-G5 |
| 分辨率下限 1024×768 | 画面自适应不溢出/不裁切 | AT-L5 |
| 加命达到生命上限（如有） | +1 是否封顶处理正确 | AT-F4 |

### 异常路径（Negative / Error）
| 场景 | 预期 | 关联用例 |
|------|------|----------|
| 基地被己方/任意子弹命中 | 立即失败，不区分来源 | AT-E4 |
| 双方子弹对撞 | 双弹抵消，不穿透继续伤害 | AT-E3 |
| 坦克顶向墙体/坦克/水 | 被正确阻挡，无穿模 | AT-B3 / AT-E5 |
| 切后台 / 失焦 | 自动暂停，回前台不暴走跳帧 | AT-H4 |
| localStorage 不可用 / 已满 / 被禁用 | 不崩溃，游戏仍可玩（仅不持久化） | AT-K3 / AT-L7 |
| 清除本地数据后重开 | 重置且不报错 | AT-K3 |
| 运行时异常 | 捕获并友好提示，不整页白屏 | AT-L7 |
| 触屏关闭时误触虚拟键 | 不影响键盘操作 | AT-L5 |

### 评审修订追踪
- 评审意见「游戏结束次数字体改大 2 个号」→ 唯一可量化落点为**结束结算（FR-8 / 游戏结束画面）的总分字体相对已评审原型再放大 2 个字号**，由 **AT-G2** 与 Gherkin「游戏失败结算与最高分」场景验收。
- 验收方式：与原型基线视觉对比，确认结算分数字号确实增大 2 个等级且不破坏结算面板布局（不溢出、不遮挡「再来一局/返回主菜单」按钮）。

> 备注（不纳入本期验收）：OUT-1~OUT-8（联网对战、账号、内购、云榜单、关卡编辑器、原生 App、小屏手机深度适配、3D）。验收过程中若发现相关功能，视为越界，不作为通过条件。
